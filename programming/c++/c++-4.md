# 4 构造函数

1.拷贝构造，其实就是拷贝我们的构造函数。 2.关键字：const & 3.本质：本质上是构造函数，参数是**类的常引用** 4.如果不写那就是默认拷贝构造，默认的拷贝构造函数，会逐个赋值非静态成员\(浅复制，浅拷贝\)，复制的是值。

## 如果调用了拷贝构造，那么就不会调用构造函数了。那什么时候会调用拷贝构造呢？

1.新建一个对象，给现有的对象进行初始化。

```cpp
#include <iostream>
using namespace std;

class CStu
{
public:
    CStu() {}
    CStu(const CStu&amp; stu) {}
};

int main()
{
    CStu a;
    CStu b(a); // 方式一
    CStu c = a;// 方式二
    CStu d = CStu(a); // 方式三，通过临时对象去创建
    CStu* e = new CStu(a);// 方式四
    system("pause");
    return 0;
}
```

如果是赋值是不会的。

```text
    CStu a;
    CStu b;
    b = a; //不会调用拷贝函数。
```

2.函数参数传递的是对象的值的时候，其次是作为范围值的时候

```text
方式一：
void fun(CStu stu){}
方式二：
CStu fun(){
    CStu stu； // 创建临时对象，类似方式三。
    return stu; // 返回对象
}
```

## 深拷贝和浅拷贝

1.浅拷贝的话就是只**拷贝值，注意如果是指针的话**，指针变量装的是一个地址，这个值就是地址。所以当我们delete释放的话就会重复释放一段空间。 2.当指针成员的时候我们就不能使用浅拷贝了，不能直接赋值，要用内存拷贝。也就是深拷贝。

```cpp
    //如果是浅拷贝，直接复制指针的值，也就是装的地址。
    CStu(const CStu&amp; stu)
    {
        this->a = stu.a;

    }
    // 深拷贝，内存拷贝
    CStu(const CStu&amp; stu)
    {
        this->a = new int[2];
        memcpy(this->a, stu.a, 8);
    }
```

## 内联函数

关键字：inline，函数定义和声明要加inline。 **作用：用相应的代码替换调用。比常规函数调用稍快，但是要占用更多内存\(每次调用都会复制替换\)。** 需要注意的是，这是编译器的特性。有些编译器支持内联函数，有些不支持。如果不支持则写了也没用，也是当作普通的函数去处理。

类内默认是内联函数，类外不是

```cpp
class Stu
{
public:
    void fun() //默认是内联函数
    {}
}

// 不是内联函数
void Stu:: fun1()
{}
```
